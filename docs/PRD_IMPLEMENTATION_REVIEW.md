# PRD 实现与风格对齐检查（APP + Web）

## 检查范围
- PRD 文档：`docs/PRD.md`
- APP：`app/lib/screens/*`
- Web：`web/app/*`

## 1) PRD 功能实现状态（当前代码）

### 已有基础实现
- 五大底部导航结构（首页/探索/发布/市场/我的）已在 APP 实现。
- 探索模块已有院校、专业、导师三类基础信息展示。
- 市场模块已有文旅、艺术品、资源对接三个入口。
- 个人中心已有申请进度、收藏、作品占位及基础身份信息展示。
- Web 端已覆盖对应页面路由，形成与 APP 基本一致的模块入口。

### 仍属于 MVP 阶段（与 PRD 的差距）
- 智能匹配引擎、AI 文书/作品集助手尚未接入真实能力。
- 院校详情页的深度信息（对比、录取数据可视化）尚未落地。
- 社区问答与消息通知仍为静态展示，未形成完整互动链路。
- 交易保障、订单、合同、支付流程未打通。
- 认证体系目前仅为前端呈现，未联动后端身份校验。

## 2) 与主流咨询类产品风格对比（主要差异）

参考风格：BetterHelp / Talkspace / Calmerry / 常见咨询 SaaS 门户。

原有差异：
- 首页存在“演示组件感”内容（非业务主线），转化路径不聚焦。
- 身份可信度信息不够突出（认证、响应时间、可预约资源）。
- 部分页面文案偏“艺术展示”而非“咨询决策支持”。
- Web 个人页收藏模块使用动态 Tailwind 类名，存在样式失效风险。
- 登录后在 APP 直接进入单页，不符合主导航流转。

## 3) 本次已完成的迁移与优化

### APP
- 修复登录后路由，进入 `MainScreen`，保持五 Tab 主流程完整。
- 首页新增咨询信任区块（认证导师、本周可约、平均回复）。
- 首页新增 1v1 咨询 CTA 卡片，强化转化入口。
- 个人中心新增“身份与可信度”卡片（认证标签 + 资料完整度）。

### Web
- 首页移除演示计数器与配色展示，替换为咨询导向首页结构：
  - Hero + 双按钮主行动
  - 平台信任数据
  - 热门院校
  - 咨询案例与最新资讯
- 个人中心补充身份信息区，强化用户可信信息呈现。
- 修复收藏模块动态 Tailwind 类名，改为显式 class，避免构建时丢样式。
- Dashboard 品牌与指标文案向咨询业务语义对齐。
