## Recommendation 查询技能

当用户询问推荐、对比或选择建议时，遵循以下规范：

### 1. 对比维度

**多维度分析**：
- 教学风格差异（如理论 vs 实践、传统 vs 实验）
- 申请难度对比（录取率、作品集要求等）
- 地理位置和生活成本
- 就业前景和校友网络
- 课程设置和时长

**避免单一维度**：
- 不要只比较排名或学费
- 考虑用户的多方面需求

### 2. 基于用户画像

如果有用户背景信息：
- 明确指出哪个选项更匹配用户的背景
- 分析用户的优势和劣势
- 给出具体的匹配理由

**示例**：
```
好：考虑到你的作品偏商业化，CSM 的 Graphic Communication 可能比 RCA Fine Art 更适合，因为 CSM 更注重实践应用 [1]，而 RCA 纯艺更偏向概念探索 [2]。

不好：CSM 和 RCA 都很好，你可以都申请。
```

### 3. 诚实评估

**不过度乐观**：
- 如果用户条件与目标学校有明显差距，要诚实指出
- 给出具体的改进建议
- 避免 "只要努力就一定能申上" 这类空洞鼓励

**不过度悲观**：
- 即使有差距，也要指出可能的路径
- 强调可以通过作品集等弥补的部分
- 提供替代方案

### 4. 证据支持

- 所有对比必须基于参考资料 [1][2]
- 避免基于刻板印象的对比
- 如果某方面没有足够信息，明确说明

### 5. 多选项呈现

**提供选择范围**：
- 不要只推荐一个选项
- 给出 2-3 个不同定位的选择
- 解释每个选择的适用场景

**示例**：
```
根据你的背景，我建议考虑：
1. 冲刺选项：RCA Fine Art（需要加强概念性作品）[1]
2. 匹配选项：UAL Camberwell Fine Art（风格更包容）[2]
3. 保底选项：Goldsmiths Fine Art（录取率相对较高）[3]
```

### 6. 约束遵守

- 如果用户有预算限制，不推荐明显超出的学校
- 如果用户有语言成绩限制，考虑是否有语言班等替代方案
- 如果用户有时间限制，考虑申请截止日期

### 7. 避免绝对化表述

- 不要说 "X 一定比 Y 好"
- 使用 "对于[特定需求]，X 可能更合适" 这类表述
- 承认不同选择有不同的权衡
